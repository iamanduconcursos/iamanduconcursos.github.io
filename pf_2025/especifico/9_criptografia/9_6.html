🔐 Resumo Criptografia – Hashes (Perito Digital PF)

Conceito-chave: Funções hash criptográficas transformam dados de entrada (mensagem) em um valor fixo (resumo), usado para garantir integridade e autenticidade.

📘 Propriedades Desejáveis de Funções Hash

  • Determinística: mesma entrada sempre gera a mesma saída
  • Rapidez: computação eficiente
  • Resistência à colisão: difícil encontrar duas entradas diferentes com o mesmo hash
  • Resistência à pré-imagem: difícil encontrar uma entrada a partir do hash
  • Resistência à segunda pré-imagem: difícil encontrar outra entrada com o mesmo hash de uma entrada dada

📊 Comparativo dos Principais Algoritmos

Algoritmo Comprimento do Hash Status Vulnerabilidades Uso Atual
MD5 128 bits Obsoleto Colisões práticas Baixo
SHA-1 160 bits Depreciado Colisões comprovadas Baixo
SHA-2 (SHA-256, SHA-512) 256 a 512 bits Ativo Sem colisões conhecidas Alto
SHA-3 Variável (224, 256, 384, 512) Ativo (Padrão NIST) Base diferente (Keccak) Crescente

🔍 Detalhamento dos Principais Hashes

🔸 MD5

  • Hash de 128 bits
  • Alta velocidade
  • Obsoleto: colisões geradas com baixo custo computacional
  • Não recomendado para segurança

🔸 SHA-1

  • Hash de 160 bits
  • Colisões comprovadas (ataque SHAttered, 2017)
  • Não é seguro
  • Foi amplamente usado em certificados SSL

🔸 SHA-2

  • Família: SHA-224, SHA-256, SHA-384, SHA-512
  • SHA-256 é o mais utilizado
  • Considerado seguro
  • Usado em criptomoedas (Bitcoin), SSL, VPNs

🔸 SHA-3 (Keccak)

  • Baseado em construção esponja (sponge construction)
  • Substitui internamente SHA-2, mas não é uma atualização direta
  • Mais resistente a alguns ataques teóricos

💥 Colisões – Conceito e Impacto

Colisão: Quando duas entradas diferentes geram o mesmo valor hash.
  • Impacto: Permite falsificação de mensagens digitais, fraudes, burlas em verificações de integridade.
  • CEBRASPE cobra: Exemplo clássico é o ataque de colisão ao MD5 ou SHA-1 → resultado: documento falso com hash idêntico ao legítimo.

⚠️ Questões Estilo CEBRASPE

  • “SHA-1 ainda é seguro para autenticação digital.” – FALSO
  • “SHA-3 é derivado diretamente do SHA-2.” – FALSO
  • “Funções hash criptográficas devem ser reversíveis.” – FALSO
  • “MD5 e SHA-1 não são recomendados para novos sistemas.” – VERDADEIRO

🧠 Dicas-Chave para a Prova

  • Evite MD5 e SHA-1 em qualquer aplicação moderna
  • SHA-256 (SHA-2) é padrão atual mais seguro
  • SHA-3 usa construção esponja e é mais novo, mas ainda em fase de adoção
  • Colisões comprometem autenticidade → essencial entender para questões de segurança
