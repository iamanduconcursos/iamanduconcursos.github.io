🗃️ Banco de Dados NoSQL - Resumo Estratégico

Estudo direcionado para a prova de Perito Digital da PF e no estilo da banca CESPE/Cebraspe.

🔎 1. O que é NoSQL?

NoSQL refere-se a uma classe de bancos de dados que não seguem o modelo relacional tradicional (SQL).

  • Projetado para alta escalabilidade e grande volume de dados.
  • Normalmente não usa JOINs e não exige esquema fixo.
  • Não significa “sem SQL”, mas sim “não apenas SQL”.
✅ Indicado quando há alta complexidade, volume ou necessidade de performance em leitura/gravação.

📂 2. Tipos de Banco NoSQL

Tipo Descrição Exemplos Casos de Uso
Chave-Valor Armazena dados como pares (chave, valor) Redis, DynamoDB Cache, sessões, perfis simples
Documentos Armazena documentos (geralmente JSON) MongoDB, CouchDB Aplicações web, APIs REST
Colunar Armazena dados por colunas (não por linhas) Cassandra, HBase Big Data, BI, Analytics
Grafos Armazena nós e arestas (relacionamentos) Neo4j, ArangoDB Redes sociais, recomendações

⚙️ 3. Características do NoSQL

  • Escalabilidade horizontal: fácil de distribuir entre servidores.
  • Alta disponibilidade: resistente a falhas.
  • Baseado em CAP Theorem: Consistência, Disponibilidade, Tolerância à Partição.
  • Desnormalização: evita joins para performance.
⚠️ CESPE ADORA: perguntas sobre o Teorema CAP (Consistency, Availability, Partition Tolerance).

📚 4. Diferenças SQL vs. NoSQL

Aspecto Relacional (SQL) NoSQL
Modelo Tabelas com colunas fixas Documentos, grafos, colunas, chave-valor
Esquema Fixado e declarado Flexível ou inexistente
Joins Sim Evita joins
Escalabilidade Vertical (aumenta servidor) Horizontal (mais servidores)
Transações ACID Eventual consistency

🧠 5. Pegadinhas CESPE / Dicas de Prova

  • ❌ “NoSQL é melhor que SQL.” — Errado! (depende do uso)
  • ✅ “NoSQL permite escalabilidade horizontal.” — Certo!
  • ❌ “MongoDB usa tabelas e colunas.” — Errado! (usa documentos JSON)
  • ✅ “Sistemas NoSQL não exigem um esquema fixo.” — Certo!
  • ✅ “Bases NoSQL favorecem a performance em grandes volumes de dados.” — Certo!

🧪 6. Aplicações práticas

  • Big Data: Cassandra, Hadoop
  • Web em tempo real: Firebase, Redis
  • Social Graphs: Neo4j
  • APIs JSON: MongoDB

📌 7. Resumo Mental Rápido

  • NoSQL = “Not Only SQL”
  • Documentos = MongoDB
  • Chave-Valor = Redis
  • Colunar = Cassandra
  • Grafos = Neo4j
  • Escalabilidade horizontal = marca registrada
  • Eventual consistency > ACID (em alguns cenários)
