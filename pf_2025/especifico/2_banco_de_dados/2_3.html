💾 SQLite - SGBD Relacional

Resumo estratégico da disciplina Bancos de Dados com foco na banca CESPE/Cebraspe para o concurso de Perito Digital da Polícia Federal.

📌 1. O que é SQLite?

  • É um SGBD relacional leve e embutido.
  • Não exige instalação de servidor: zero configuração.
  • Armazena o banco de dados em um único arquivo.
  • Extremamente usado em apps móveis, IoT e sistemas embarcados.
✅ Dica CESPE: SQLite é um banco relacional e transacional, mesmo sem servidor.

🔍 2. Características principais

  • Servidor embutido: biblioteca que é integrada à aplicação.
  • ACID: Garante propriedades de atomicidade, consistência, isolamento e durabilidade.
  • SQL padrão: Suporta a maior parte da linguagem SQL.
  • Tipos de dados dinâmicos: Tipagem é baseada em valores, não em colunas (tipagem fraca).

📂 3. Vantagens

  • Pequeno (menos de 1 MB)
  • Portável: funciona em quase qualquer SO
  • Ideal para testes, protótipos e pequenos sistemas
  • Baixo consumo de recursos
🧠 Exemplo: Aplicativos Android e iOS frequentemente usam SQLite como banco local.

🚫 4. Limitações

  • Não suporta múltiplas conexões simultâneas com alta concorrência.
  • Não recomendado para sistemas corporativos com grande volume de usuários.
  • Sem recursos avançados de segurança nativos (como autenticação por usuários).

🛠️ 5. Comandos SQL no SQLite

Comando Descrição Exemplo
CREATE TABLE Cria uma tabela CREATE TABLE clientes (id INTEGER PRIMARY KEY, nome TEXT);
INSERT INTO Insere dados INSERT INTO clientes (nome) VALUES ('João');
SELECT Consulta dados SELECT * FROM clientes;
UPDATE Atualiza dados UPDATE clientes SET nome = 'Maria' WHERE id = 1;
DELETE Remove registros DELETE FROM clientes WHERE id = 1;

⚠️ 6. Pegadinhas comuns da CESPE

  • ❌ “SQLite não é relacional.” — Errado!
  • ❌ “SQLite exige instalação de servidor.” — Errado!
  • ✅ “SQLite é um banco transacional que suporta ACID.” — Certo!
  • ❌ “SQLite usa tipos de dados rígidos como no PostgreSQL.” — Errado!
  • ✅ “SQLite é baseado em arquivo e funciona bem para aplicações locais.” — Certo!

🧠 7. Fatos Rápidos

  • Extensão de arquivo padrão: .sqlite ou .db
  • Usado por: Firefox, Chrome, Android, Skype, Dropbox
  • Não possui gerenciamento de usuários integrado
  • Permite transações com BEGIN, COMMIT e ROLLBACK

📌 8. Quando usar SQLite

  • Aplicativos mobile
  • Sistemas desktop leves
  • Dispositivos embarcados (IoT)
  • Ambientes de teste/protótipo
✅ CESPE valoriza o entendimento da aplicabilidade e limitações reais da tecnologia.
