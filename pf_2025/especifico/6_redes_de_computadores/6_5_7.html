🌐 Protocolos de Rede (Resumo para Concurso PF)

Resumo essencial da disciplina Redes de Computadores com foco no concurso de Perito Digital da Polícia Federal. Estruturado para te dar o máximo de conteúdo com o mínimo de leitura. Ideal para questões da banca CESPE/Cebraspe.

📦 1. Camada de Rede

  • IP (Internet Protocol): Endereçamento e roteamento de pacotes.
  • IPv4: 32 bits (ex: 192.168.0.1)
  • IPv6: 128 bits (ex: 2001:0db8::7334)
  • ARP: Mapeia IP para MAC (camada de enlace ↔ rede).
  • ICMP: Diagnóstico e mensagens de erro (ex: ping).
  • DNS: Tradução de nomes para IPs (ex: www.exemplo.com → 8.8.8.8).
  • DHCP: Atribui IPs dinamicamente a clientes na rede.

🔀 2. Camada de Transporte

  • TCP: Orientado à conexão, confiável (confirmação, retransmissão).
  • UDP: Sem conexão, mais rápido, mas sem garantia de entrega.
  • SCTP: Multistreaming + confiável, combina TCP e UDP.

🌐 3. Camada de Aplicação

  • HTTP: Transmissão de páginas web (porta 80).
  • HTTPS: HTTP com criptografia TLS/SSL (porta 443).
  • FTP: Transferência de arquivos (portas 20 e 21).
  • SFTP: FTP com segurança via SSH (porta 22).
  • SSH: Acesso remoto seguro (porta 22).
  • SMTP: Envio de e-mails (porta 25).
  • POP3: Recebimento de e-mails, baixa para o cliente (porta 110).
  • IMAP: Acesso remoto ao e-mail, sincronização com servidor (porta 143).

🎯 5. Dicas CESPE/Cebraspe

  • 📬 POP3 baixa e-mails para o cliente e apaga do servidor (salvo configuração).
  • 🔁 IMAP mantém os e-mails no servidor, ideal para múltiplos dispositivos.
  • 🔐 SFTP é diferente de FTP com SSL (FTPS). Ele usa SSH.
  • 🚨 ICMP não é confiável para segurança (pode ser usado para ataques).
  • 📥 DHCP oferece IP, gateway, DNS automaticamente.
  • 🧠 DNS usa geralmente UDP na porta 53, mas pode usar TCP em transferências zone.
💡 Resumo mental:
HTTP = 80 | HTTPS = 443 | FTP = 21 | SFTP = 22
SMTP = 25 | POP3 = 110 | IMAP = 143

🧪 6. Aplicações Práticas para Perícia Digital

  • Identificar protocolos e portas em logs de tráfego
  • Reconstruir sessões HTTP/S ou FTP
  • Detectar acessos remotos via SSH
  • Rastrear remetentes de e-mails via cabeçalhos SMTP
