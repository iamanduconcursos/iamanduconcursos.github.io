Ataques a Redes de Computadores

Resumo para Prova - Segurança de Redes - CEBRASPE/CESPE - Perito Digital PF

1. Ataques DoS e DDoS

  • DoS (Denial of Service): Ataque que visa indisponibilizar um serviço ou recurso, sobrecarregando-o com requisições.
  • DDoS (Distributed DoS): Ataque DoS distribuído, realizado por múltiplos dispositivos infectados (botnets), aumentando o volume e dificuldade de defesa.

2. Botnets

  • Redes de computadores/zumbis infectados controlados por um atacante para realizar ataques coordenados, como DDoS, envio de spam, roubo de dados, etc.
  • Usam canais de comando e controle (C&C) para comunicação e controle remoto.

3. Phishing

  • Técnica de engenharia social para obter dados sensíveis, como senhas e informações financeiras, por meio de mensagens falsas (e-mails, sites, etc.).
  • Explora confiança e induz usuário a entregar informações.

4. Zero-Day Exploits

  • Exploração de vulnerabilidades desconhecidas pelo fabricante e para as quais ainda não há patch ou solução.
  • Extremamente perigosos por serem inesperados e não detectados por antivírus tradicionais.

5. Ataques por Flood

  • Ping da Morte: Envio de pacotes ICMP malformados ou oversized que causam falha no sistema alvo.
  • UDP Flood: Envio massivo de pacotes UDP para sobrecarregar rede ou sistema, consumindo recursos.

6. Ataques em Camada de Rede e Enlace

  • MAC Flooding: Envio de frames com múltiplos endereços MAC falsos para esgotar a tabela de switches e causar broadcast de tráfego.
  • IP Spoofing: Falsificação do endereço IP de origem para mascarar a origem do ataque ou burlar sistemas de segurança.
  • ARP Spoofing (ou ARP Poisoning): Envio de mensagens ARP falsas para associar o endereço MAC do atacante ao IP de outro dispositivo, permitindo interceptação de tráfego (Man-in-the-Middle).

7. Exploração de Vulnerabilidades e Injeções

  • Buffer Overflow: Exploração de falha em programas que não controlam corretamente o tamanho de dados, permitindo execução de código malicioso.
  • SQL Injection: Inserção de comandos SQL maliciosos em entradas de dados para manipular ou acessar banco de dados indevidamente.
  • Cross-Site Scripting (XSS): Injeção de scripts maliciosos em páginas web para roubo de cookies, sessões e ataques ao usuário final.

8. DNS Poisoning (Envenenamento de Cache DNS)

  • Manipulação maliciosa do cache DNS para associar nomes de domínios a endereços IP falsos.
  • Usado para redirecionar usuários a sites fraudulentos (phishing) ou interceptar tráfego.
